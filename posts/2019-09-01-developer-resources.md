## Project Documentation

Before you start a new project, it's generally a good idea to take time to write down a README. This is a practice known as [Readme Drive Development][rdd].

- _Yegor Bugayenko_ has a crash course in writing [Elegant READMEs][elegant-readme]

## Architecture

Another important area to spend time on is to figure out your code architecture.

- Read up on [Application Holotypes][application-holotypes] and proposed architecture decisions by _Jason Miller_.

- _Martin Fowler_ has some great motivators to answer the question: [Is High Quality Software Worth the Cost?
  ][quality-software].

## Web application development

- Tips for [Designing Native-Like Progressive Web Apps For iOS][native-link-pwa] from _Appscope_.

### Styling and CSS

- A set of general purpose [CSS rules that will make your life easier][css-rules] by _Nick Gard_.

- [Fluid Responsive Typography With CSS Poly Fluid Sizing][fluid-typography]

- A library of [Beautifully simple click-to-copy CSS effects][css-effects] (buttons, text, input, loaders).

## APIs

- [Brandfetch][brandfetch-api]: personalization API to grab logos, colors, and images from brand websites.

## Tools

- Carbon is a web app that let's you create and share [beautiful images of your source code][carbon].

- _Whimsical_ is the cleanest and prettiest way to make [wireframes, flowchats, and mindmaps][whimsical].
